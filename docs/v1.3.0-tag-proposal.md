## Versioning note

A new public function (`tool_lorentz_transform`) plus a new UI tab and a
new MCP tool is new backward-compatible functionality → **minor
(1.3.0)** by semver. (An earlier internal draft weighed 1.2.1-as-patch;
1.3.0 is the strict-semver choice and was taken deliberately with the
user.)

## What v1.3.0 contains

- **`tool_lorentz_transform`** (tool layer + MCP server): boost/rotation
  + axis → Lorentz matrix Λ = ZZ* over the Σ basis, evaluated
  numerically. Axis normalized; numbers-and-enums only (K26). A test
  PINS it to the symbolic engine (`lorentz_matrix ∘ boost_alpha /
  rotation_alpha`) at unit axes so the fast path cannot drift. Exposed
  as the MCP tool `lorentz_transform` (the `test_fastmcp_wiring_smoke`
  exact-set now lists five tools).
- **"Lorentz transform" UI tab**: a Radio (boost/rotation), the
  parameter (φ/θ, radians) and an axis as `gr.Number` inputs, a compute
  button; outputs the covariance parameters α (computed, never typed),
  Λ (4×4 real), and an honest validity badge — ΛᵀgΛ = g residual,
  det Λ = +1, orthochronous (Λ₀₀ ≥ 1), imaginary-leak.
- Overflow-safe from the start: a valid-but-extreme boost rapidity is a
  readable "angle too large" reason, not a traceback (numpy overflow +
  a finiteness check), and never serves an inf matrix.
- Tests 399 → **411** (the 8 tool tests + the overflow regression run
  everywhere; the 3 UI-callback tests gate on the `ui` extra; the MCP
  exact-set test was updated, not added).
- Security posture unchanged: 127.0.0.1 + share=False constants intact;
  no new file/parse surface; α never enters as text anywhere.
