# v2.0 Version-Tag PROPOSAL

**Status: PROPOSAL — not applied.** This document proposes a release tag
for Organon v2. It does **not** create one. Tagging a release is a
user decision (it is a public, standing marker on the repository), and it
sits alongside the other gated actions listed at the end of this file. No
`git tag` command was run in producing this document.

## Proposal

Apply an annotated tag **`v2.0.0`** to the current `main` HEAD once the
Stage 22 closure commit is in place — i.e. after the commit that adds this
file, the retrospective, and the Stage 22 report.

Semantic-versioning read: this is the first *released* state of the v2
line, so the major version is 2 and the minor/patch are 0.0. There is no
prior v2.x tag to increment from; v2 has been developed under the
FROZEN-22 roadmap rather than tagged incrementally.

## Why v2.0 now

The closure criteria the project set for itself are all met at the point
this proposal is written:

- All 22 FROZEN-22 stages are complete (A0–A21 closed; A22 is this closure
  stage). The stage count never changed; the two scope revisions (static
  web viewer, English project language) were reasoned in writing and left
  the count invariant. See `docs/retrospective-v2.md`.
- 288 tests green; CI green on the three-version matrix (Python
  3.10/3.11/3.12). The six-layer verification contract
  (`docs/VERIFICATION.md`) is intact and was stress-tested from the
  outside in A21.
- No open acceptance debt: the only deferred item (the guarded-atoms
  second half) was discharged in A20 as a verified negative result.
- The outward deliverables (the Kuntman feedback package, the
  candidate-findings inventory, the report generator, the MCP and web
  surfaces as code+tests) are internally consistent, each mathematical
  statement carries its evidence class, and nothing is hosted or exposed.

## Suggested annotated-tag message (draft)

The user may edit this freely; it is a starting point, not a fixed string.

```
v2.0.0 — Organon v2 (organon-mueller)

First released state of the v2 line. Experimental Stokes–Mueller reasoning
engine: a discovery engine (equality saturation over the covariance-vector
term language, every candidate certified by exact symbolic proof) and a
decomposition deriver (the symmetry-conditioned decompositions of the
Kuntman–Arteaga papers derived from rank-1 minor conditions), extended to
the coupled-dipole scattering results, all under a six-layer verification
contract.

- 22/22 FROZEN-22 stages complete; 288 tests; CI green on Python 3.10–3.12.
- Results labelled verified (machine-checked against the published tables)
  or candidate (beyond the tables — no novelty or physics claim is made;
  interpretation is reserved for domain experts, novelty-protocol step 5).
- Experimental research software. No licence is set (deliberate; a user
  decision). Nothing is hosted or exposed.
```

## The command — for the user to run, if and when they choose

Not executed here. If the user decides to tag, the commands would be:

```bash
# on the Stage 22 closure commit:
git tag -a v2.0.0 -m "v2.0.0 — Organon v2 (organon-mueller)"   # or an edited message / -F file
git push origin v2.0.0
```

## Boundary

Tagging `v2.0.0` remains the user's decision, together with the other
gated actions that this closure does not take: sending the Kuntman
feedback package, filing the egglog upstream issue, hosting the MCP
server, exposing the web viewer, and setting a repository licence. The
assistant produced the proposal; the user takes (or declines) the action.
