# v1.0.0 Version-Tag PROPOSAL

**Status: PROPOSAL — not applied.** This document proposes a release tag
for `organon-mueller`. It does **not** create one. Tagging a release is a
user decision (it is a public, standing marker on the repository), and it
sits alongside the other gated actions listed at the end of this file. No
`git tag` command was run in producing this document.

> **Supersedes** `docs/v2.0-tag-proposal.md` (Stage 22). The original
> proposal suggested `v2.0.0`, inheriting the number from the "Organon v2"
> program-generation name. **Naming decision (user, 2026-07-16):**
> `organon-mueller` is a separate project from
> [Organon v1](https://github.com/tanzercakir-commits/Organon) (which owns
> the `v1.x` line *in its own repository*), so this package's semantic
> versioning starts at **`v1.0.0`**. "Organon v2" remains the *narrative*
> name for this generation of the research program; the *package/release*
> version line is `1.x`. The Stage 22 spec and report are kept unchanged
> as historical records of the original proposal.

## Proposal

Apply an annotated tag **`v1.0.0`** to `main` once the local-UI milestone
(UI-1) lands — i.e. the first release contains the complete engine (the
closed FROZEN-22 roadmap, commit `8a8d760`) **plus** the local web
interface, so the first thing users install is also the first thing that
is easy to use.

Alternative (if the user prefers the engine-only closure marked
separately): tag the closure commit `8a8d760` as `v1.0.0` and the UI
milestone as `v1.1.0`. Both are sound; the single-tag option is simpler
and is the recommended default.

## Why v1.0.0 now

- All 22 FROZEN-22 stages are complete; the verification contract
  (`docs/VERIFICATION.md`) is intact and was stress-tested externally in
  A21. See `docs/retrospective-v2.md`.
- Full suite green; CI green on the three-version matrix (Python
  3.10/3.11/3.12).
- No open acceptance debt (the guarded-atoms second half was closed in
  A20 as a verified negative result).
- The package metadata version (`pyproject.toml`) is aligned to `1.0.0`.
